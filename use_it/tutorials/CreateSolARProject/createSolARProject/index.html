Instructions

Initialize your development project

Open QTCreator and create a new project (in File menu). Choose "Non-Qt project" and "Plain C++ Application", set the location and the name of your project (for example SolARFiducialTutorial), and select as build system "qmake", and then the building kit of your choice. Finally, click on finish to create your project. Now, you need to select the file called "SolARFiducialTutorial.pro" in your project tree, and you have to add the following line before defining your source files:

DEPENDENCIESCONFIG = sharedlib
include ($$(BCOMDEVROOT)/builddefs/qmake/templateappconfig.pri)

Be sure to have defined your environment variable $BCOMDEVROOT$ and to have copied in this folder all the files defining your build pipeline downloadable here. If not, download the zip file, unzip it, and copy the content of the folder builddefs-qmake-solar_v1.0.0 in $BCOMDEVROOT$/builddefs/qmake/.

Then, create in your project folder a file called packagedependencies.txt and add in it your dependencies, namely the basic dependencies required by the SolARFramework (SolARFramework, xpcf, boost, eigen and spdlog) as well as the one required for the openCV module (SolARModuleOpencv and opencv):

SolARFramework|1.0.0|SolARFramework|bcomBuild|url_repo_artifactory
SolARModuleOpenCV|1.0.0|SolARModuleOpenCV|bcomBuild|url_repo_artifactory
xpcf|1.0.0|xpcf|thirdParties|http://repository.b-com.com/
boost|1.64.0|boost|thirdParties|http://repository.b-com.com/
opencv|3.2.0|opencv|thirdParties|http://repository.b-com.com/
spdlog|1.0.0|spdlog|thirdParties|http://repository.b-com.com/
eigen|3.3.4|eigen|thirdParties|http://repository.b-com.com/amc-generic

Now you can run QMake (click right of your project in QT creator, and click on run QMake). Every pathes for headers and library links are now set.
