package & download third parties

packaging

  1. clone the following git repository: git clone ssh://gitolite@forge.b-com.com/bcom-templates/builddefs/builddefs-scripts.git

Starting from a fresh installed package (ie. result of a "make install"), run the following perl script (under builddefs-scripts/xplatform)

$ bcom-packager.pl
  OPTIONS:
      -s, --sourcedir                  => product root directory (where libs and includes are located)
      -o, --osname                     => specify the operating system targeted by the product build. It is one of [win|mac|linux]. (defaults to the current OS environment)
      -i, --includedir                 => relative path to include folder to export (defaults to the sourcedir provided with -s)
      -l, --libdir                     => relative path to the library folder to export (defaults to the sourcedir provided with -s)
      -r, --redistfile                 => relative path and filename of a redistribution file to use (such as redist.txt intel ipp's file). Only listed libraries in this file will be packaged
      -d, --destinationdir             => package directory root destination (where the resulting packaging will be stored)
      -p, --packagename                => package name
      -v, --packageversion             => package version
      -n, --ignore-mode                => forces the pkg-config generated file to ignore the mode when providing -L flags
      -m, --mode [debug|release]       => specify the current product build mode. Binaries will be packaged in the appropriate [mode] folder
      -a, --architecture [x86_64|i386] => specify the current product build architecture. Binaries will be packaged in the appropriate [architecture] folder
      -w, --withsuffix suffix          => specify the suffix used by the thirdparty when building with mode mode
      -u, --useOriginalPCfiles         => specify to search and use original pkgconfig files from the thirdparty, instead of generating them
      -h, --help                       => display this help

  EXAMPLES:
      -s {path_to_root_sourcedir} --o win d {path_to_destination_dir} -p intel-tbb -v 2017.5 -l build/macos_intel64_clang_cc8.0.0_os10.11.6_release -m release -i include

  1. Use the artifactory packager scripts.

Under windows, use builddefs-scripts/win/artiPackager.bat

Under linux, use builddefs-scripts/unixes/artiPackager.sh

Usage: run the .bat / .sh scripts where the packages are.

Using pkgm-bcom

  1. Source: https://forge.b-com.com/www/bcom-templates//tools/pkgm-bcom-index/

  2. You must have java installed on your machine: http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html

  3. You will need to create an artifactory API_KEY: http://repository.b-com.com/webapp/#/profile

  4. Download pkgm-bcom tool here: http://repository.b-com.com/bcom-devtools-generic-local/pkgm-bcom/1.0.0/multi/pkgm-1.0.0-fat.jar

  5. In order to download dependencies described in a packagedependencies.txt file, you will need to launch:

    java -jar [path to]/pkgm-1.0.0-fat.jar install -a x86_64 -c release -m shared -f packagedependencies.txt -k [YOUR artifactory API KEY]

  6. NB: this will work only if third parties are available on artifactory repository

For example packagedependencies.txt can include the two following lines:

opencv|3.2.0|opencv|thirdParties|http://repository.b-com.com/argo-generic
boost|1.64.0|boost|thirdParties|http://repository.b-com.com/argo-generic
