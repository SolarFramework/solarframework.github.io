Architecture

SolAR framework is conceived to help you to connect and assemble different software components.

It is then possible to choose the best components for your usage.

SolAR software priority

SolAR has been conceived to offer the best management of components to allow:

  • low memory usage

  • dynamic component loading

  • easy building

  • modularity

For this, SolAR framework relies on a Component Manager conceived by the project team.

Specification

Best of vision tools

The best of vision tools should be used, and as the best components are based on C++, this framework also.

Modularity

SolAR framework is conceived to be modular, so that you can add several components, and load it when needed.

Framework API

SolAR framework can be used directly in C using its API, for C developers.

To help developsers, components are documented :

  • API documentation

  • components sample documentation

  • tutorials
