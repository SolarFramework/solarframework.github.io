Example.xml
<?xml version="1.0" encoding="UTF-8" standalone="yes" ?>
<xpcf-registry autoAlias="true">

<module uuid="xxxx" name="xxxx" path="xxxx" description="xxxx">
  <component uuid="xxxx" name="xxxx" description="xxxx">
    <interface uuid="xxxx" name="xxxx" description="xxxx"/>
    <interface uuid="xxxx" name="xxxx" description="xxxx"/>
  </component>
</module>

<factory>
    <bindings>
      <bind interface="xxxx" to="xxxx" />
    </bindings>
 </factory>

<properties>
  <configuration component="xxxx">
    <property name="xxxx" type="xxxx" value="xxxx"/>
    <property name="xxxx" type="xxxx" value="xxxx"/>
  </configuration>
  <configure component="xxxx">
    <property name="xxxx" type="xxxx">
        <value>y</value>
        <value>y</value>
        <value>y</value>
    </property>
  </configure>
</properties>

</xpcf-registry>

Node

Function

Attributes
[Required by default]

module

Use that node to add a module to your pipeline.
Each module is composed of components that you can use in your pipeline.

uuid = Each module has an uuid that has to be give here.
name = The name of the module used
path = The path to the module used
description = You can describe a little bit the module function.

component

Use that node to use the component in your pipeline.
Each component inherits from interface.

uuid = Each component has an uuid that has to be give here.
name = The name of the component used
description = You can describe the component function.

interface

This node is linked to component node.

uuid = Each interface has an uuid that has to be give here.
name = The name of the interface used
description = You can describe the interface function.

factory

[Optional]
Declare a factory to manage components. Composed of bindings

bindings

Gives opportunity to put priority on components using the same interface. Composed of bind node

bind

bind a component on an interface to make priority or name on it.

interface = Interface to bind
to = target component of the bind (component of a module added)
name = [Optional] Name of the bind (link interface→component)

properties

Root Node for parameters declaration of each component

configuration

Configure component’s parameters

component = name of the component to configure

property

Change a parameter

name = name of the parameter to configure
type = string / bool / Float / Integer / UnsignedInteger …​
value = parameter value with type respect.

value

Use to define a vector. Refer to the example above.
